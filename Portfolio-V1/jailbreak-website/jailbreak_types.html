Types of Jailbreaks

There are 3 types of jailbreaks.

Untethered, Tethered, Semi-Tethered

Introduction

When a jailbroken device is booting, it loads Apple's own kernel initially. When the device is exploited, the kernel is patched. Whether or not this persists determines ifthe device remains jailbroken after a reboot.

Untethered

An untethered jailbreak is a jailbreak where the kernal patch persists even after the device is rebooted. It does not require any assistance to boot up. The kernel will be patched without the help of a computer or an application. These jailbreaks are uncommon and take a significant amount of reverse engineering to create. The last untethered jailbreak, Houdini by Abraham Masri, was released in 2016 for all devices on iOS 10.

Tethered

A tethered jailbreak is the opposite of a untethered jailbreak, in the sense that a computer is required to boot, and the device will not boot by itself. While using a tethered jailbreak, the user will still be able to refresh the interface and kill some system process without needing to reboot but if the phone does power off in any way, it can not power on again without being jailbroken again (while connected to a computer).

Semi-tethered

A mix between an untethered and a tethered jailbreak is also an option, namely semi-tethered. This type of jailbreak allows the user to reboot, but there will not be a patched kernel. Non-jailbroken functions will still run, such as making a phone call, texting, or using an app. To be able to have a patched kernel and run modified code again, the device must be jailbroken again. Tradtionally, this was to be done using a computer but in recent years, the kernel can be patched using a application installed on the device causing the jailbreak to become "semi-untethered". Most recent jailbreaks have been the semi-untethered variants.
